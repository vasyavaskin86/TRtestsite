# SportZone - Веломагазин

Современный интернет-магазин с админ-панелью, корзиной, списком желаемого и системой отзывов.

## 🚀 Как запустить на GitHub и в облаке

Чтобы сайт работал не только у вас, но и в интернете (с сохранением авторизации и базы данных), нужно выполнить два шага: деплой бэкенда и деплой фронтенда.

### 1. Подготовка
1. Создайте репозиторий на GitHub.
2. Загрузите туда все файлы проекта (кроме `.env` и `node_modules`).

### 2. Деплой Бэкенда (Node.js + MySQL)
Мы рекомендуем использовать **Render.com** или **Railway.app**, так как они поддерживают Node.js и базы данных.

1. Подключите ваш GitHub репозиторий к Render/Railway.
2. В настройках укажите переменные окружения (Environment Variables):
   - `PORT`: 3001
   - `DB_HOST`: адрес вашей облачной БД
   - `DB_USER`: пользователь БД
   - `DB_PASS`: пароль БД
   - `DB_NAME`: имя БД
   - `JWT_SECRET`: любая секретная строка для токенов
3. После успешного деплоя вы получите URL бэкенда (например, `https://sportzone-api.onrender.com`).

### 3. Настройка Фронтенда
1. Откройте файл `js/api.js`.
2. В функции `API_BASE` замените пустую строку на URL вашего бэкенда:
   ```javascript
   // Для продакшена (GitHub + облачный бэкенд)
   return "https://sportzone-api.onrender.com"; 
   ```
3. Запушьте изменения на GitHub.

### 4. Деплой на GitHub Pages
1. В настройках вашего репозитория на GitHub перейдите в раздел **Pages**.
2. Выберите ветку `main` и нажмите **Save**.
3. Через пару минут сайт будет доступен по адресу `https://ваш-логин.github.io/имя-репозитория/`.

## 🛠 Технологии
- **Frontend**: HTML5, CSS3 (Modern UI), JavaScript (ES6+)
- **Backend**: Node.js, Express
- **Database**: MySQL
- **Auth**: JWT (JSON Web Tokens)
